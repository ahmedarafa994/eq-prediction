<objective>
Complete the gain prediction fix by removing the Tanh activation from gain_mlp, removing the dead gain_trunk_head fallback code, cleaning up train.py gradient monitoring, and updating tests.

Purpose: The gain_mlp currently has nn.Tanh() as its final layer, which attenuates gradients near the +/-24 dB bounds (93% gradient loss at 23 dB). Removing Tanh and relying solely on ste_clamp gives identity gradient flow within bounds. The gain_trunk_head is dead code (only used in the else branch when mel_profile is None, which never happens during normal training/inference with n_mels=128). Cleaning these up ensures a single, clean gain path.

Output: Clean gain prediction in MultiTypeEQParameterHead, updated gradient monitoring, comprehensive tests.
</objective>
