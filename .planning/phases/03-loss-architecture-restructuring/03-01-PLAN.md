<objective>
Wire the dual forward path (H_mag_hard for hmag_loss, H_mag_soft for spectral loss) and add Gumbel-Softmax detach during warmup. Verify already-implemented features (log-cosh, independent weights, warmup gating). Create test suite.

Purpose: The model already computes both H_mag_hard and H_mag_soft, but train.py ignores H_mag_hard and loss_multitype.py doesn't accept it. This wiring gap means hmag_loss uses soft (blended) filter types, producing noisy gradients. Fixing this gives clean param regression gradient.
Output: loss_multitype.py with dual H_mag, train.py wiring, config.yaml with warmup_epochs, test_loss_architecture.py
</objective>

From insight/loss_multitype.py log_cosh_loss (lines 22-34):
```python
def log_cosh_loss(pred, target):
    diff = pred - target
    abs_diff = diff.abs()
    return abs_diff + torch.log1p(torch.exp(-2 * abs_diff)) - math.log(2)
```
