<objective>
Wire audio reconstruction loss (spectral-domain L1) and per-band activity mask weighting. These two features are currently dead code -- MR-STFT never fires because pred_audio=None, and activity_loss is always zero because active_band_mask is never passed.

Purpose: Audio reconstruction loss provides additional gradient signal for spectral shape after warmup. Activity mask ensures inactive bands don't produce spurious gain predictions. Both are listed in the phase requirements but currently produce zero contribution.
Output: loss_multitype.py with working spectral L1 recon loss, dataset.py with active_band_mask, train.py wiring both through.
</objective>

    CONTEXT.md says "spectral-domain MR-STFT: compare H_mag * wet_spectrum vs target_spectrum" but MultiResolutionSTFTLoss.forward() expects time-domain audio (B, T) and calls torch.stft() internally. Feeding it frequency response tensors would produce incorrect results. Since the model already outputs frequency responses (H_mag), a simpler spectral L1 loss is both correct and more efficient.

    ### 1. loss_multitype.py changes

    Replace the spectral loss block (lines 464-471):

        sc_loss, log_loss = self.mr_stft(pred_audio, target_audio)

    With spectral L1 reconstruction:
    ```python
    # D-06: Spectral-domain reconstruction loss (L1 in log-magnitude space).
    # pred_audio and target_audio are (B, F) frequency response magnitudes.
    # Compares predicted EQ response vs ground truth EQ response.
    # Uses log-magnitude L1 for perceptual weighting (matches hmag_loss pattern).
    if is_spectral_active and pred_audio is not None and target_audio is not None:
        pred_safe = torch.clamp(pred_audio.float(), min=1e-6, max=1e6)
        target_safe = torch.clamp(target_audio.float(), min=1e-6, max=1e6)
        loss_spectral = F.l1_loss(torch.log(pred_safe), torch.log(target_safe))
    else:
        loss_spectral = torch.tensor(0.0, device=pred_gain.device)
    ```
