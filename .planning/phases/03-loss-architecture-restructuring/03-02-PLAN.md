<objective>
Wire the two remaining loss components: (1) active_band_mask from dataset through train.py to
loss (LOSS-06), and (2) spectral reconstruction loss using H_mag_soft vs target_H_mag (LOSS-05).

Purpose: After Plan 01, the dual H_mag path and Gumbel detach are wired. This plan completes
the loss architecture by ensuring the activity regularization actually fires (currently always
zeros out because active_band_mask is never passed) and by wiring the spectral reconstruction
loss (currently dead — pred_audio is always None).

Key design choice (D-06): The existing MR-STFT loss in loss.py expects time-domain audio
(B, T). The precomputed dataset drops raw audio after caching mel-spectrograms. Therefore,
audio reconstruction uses a spectral L1 approach: compare log(pred_H_mag_soft) vs log(target_H_mag)
— which is ALREADY computed as hmag_loss. The "spectral reconstruction" for LOSS-05 is instead
implemented as a second spectral loss term using pred_H_mag_soft * mel_profile vs target
(frequency response comparison in mel space), OR simply by ensuring the existing hmag_loss
fully covers LOSS-05 by being computed from the soft path for gradient flow.

REVISED APPROACH (per research open question A3): Since pred_audio/target_audio as time-domain
is unavailable from the precomputed cache, LOSS-05 "audio-domain reconstruction loss" is
implemented as a spectral magnitude L1 between pred_H_mag_soft and target_H_mag — using the
existing spectral_loss slot in the loss function with a direct frequency response comparison
instead of MR-STFT. This is semantically equivalent to D-06 ("compare H_mag * wet_spectrum vs
target_spectrum") when the wet_spectrum is treated as a flat reference (normalized H_mag comparison).

    PART A — loss_multitype.py: Replace MR-STFT spectral_loss with spectral magnitude L1.

    The MR-STFT (loss.py:19-20) requires time-domain audio (B, T). Our precomputed batch
    has no raw audio — only mel-spectrograms and frequency responses. Instead, we use a
    spectral magnitude L1 between pred_H_mag_soft and target_H_mag.

    This approach is semantically equivalent to D-06 ("compare H_mag * wet_spectrum vs
    target_spectrum"): both pred and target have the same wet_spectrum reference, so the
    comparison reduces to |H_pred * W - H_target * W| ∝ |H_pred - H_target|, which is
    the spectral L1 we compute. The "soft" path (Gumbel-Softmax) ensures gradients flow
    through type_probs — providing the "differentiable type gradient" purpose of LOSS-05.

    Replace the spectral_loss branch (lines 463-471):

    Current:
    ```python
    if is_spectral_active and pred_audio is not None and target_audio is not None:
        sc_loss, log_loss = self.mr_stft(pred_audio, target_audio)
        loss_spectral = sc_loss + log_loss
    else:
        loss_spectral = torch.tensor(0.0, device=pred_gain.device)
    components["spectral_loss"] = loss_spectral
    ```

    Replace with:
    ```python
    # 4. Spectral reconstruction loss (LOSS-05, D-06)
    # Uses H_mag_soft (Gumbel-Softmax path) for differentiable type gradients.
    # pred_audio and target_audio carry the frequency responses (B, F) — NOT time-domain.
    # This replaces MR-STFT (which requires time-domain audio unavailable from precomputed cache).
    # Spectral L1 in log-space: log(H_pred_soft) vs log(H_target).
    if is_spectral_active and pred_audio is not None and target_audio is not None:
        pred_spec_safe = torch.clamp(pred_audio.float(), min=1e-6, max=1e6)
        target_spec_safe = torch.clamp(target_audio.float(), min=1e-6, max=1e6)
        loss_spectral = F.l1_loss(torch.log(pred_spec_safe), torch.log(target_spec_safe))
    else:
        loss_spectral = torch.tensor(0.0, device=pred_gain.device)
    components["spectral_loss"] = loss_spectral
    ```

    Note: The self.mr_stft module stays in __init__ (kept for potential future use with
    time-domain data). Only the forward() usage is replaced.
