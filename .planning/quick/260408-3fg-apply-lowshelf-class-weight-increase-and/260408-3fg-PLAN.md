Make two targeted additions to `MultiTypeEQLoss` in `insight/loss_multitype.py`:

**A — Per-class weight multipliers for focal loss:**

Add a new constructor parameter `class_weight_multipliers: list[float] | None = None` (default None).
In `__init__`, after computing `inv_w` from `type_weights_cfg`, apply multipliers if provided:

```python
if class_weight_multipliers is not None:
    mult_tensor = torch.tensor(class_weight_multipliers, dtype=torch.float32)
    inv_w = inv_w * mult_tensor
    inv_w = inv_w / inv_w.mean()  # re-normalize so mean=1 after multiplying
```

The multipliers are applied AFTER the inverse-frequency normalization, giving additional
emphasis independent of the base distribution. This is more transparent than manipulating
`type_weights` (which also affects data generation).

**B — Gain sign penalty:**

Add `sign_penalty_weight: float = 0.0` constructor parameter. Store as `self.sign_penalty_weight`.

In `forward()`, immediately after computing `loss_gain = log_cosh_loss(pred_gain, matched_gain).mean()`,
add:

```python
# Sign penalty: penalize predictions where sign(pred) != sign(gt).
# Targets the pattern where model predicts +X dB when GT is -Y dB.
# Penalty is proportional to the absolute error so large sign flips
# cost more than small ones near 0 dB.
if self.sign_penalty_weight > 0.0:
    sign_mismatch = (pred_gain * matched_gain) < 0.0  # True where signs differ
    sign_penalty = (sign_mismatch.float() * (pred_gain - matched_gain).abs()).mean()
    loss_gain = loss_gain + self.sign_penalty_weight * sign_penalty
    components["sign_penalty"] = sign_penalty.detach()
else:
    components["sign_penalty"] = torch.tensor(0.0, device=pred_gain.device)
```

Note: The sign penalty is added directly to `loss_gain` before it is scaled by `lambda_gain`
in the total loss, so `lambda_gain` still controls the overall gain loss scale.

Also update the `type_class_weights` buffer setup comment to document that multipliers
are applied after normalization.

Do NOT change any other logic — existing focal loss formula, warmup gating, Hungarian
matcher, or spectral loss are untouched.
