# Plan: Apply QLoRA/Unsloth/DeepSpeed Techniques to IDSP EQ Estimator

## Context

Your TCN model is ~2.2M parameters — tiny compared to LLMs. The techniques need **conceptual adaptation**, not direct application. Here's how each layer maps to your audio model:

| Technique | LLM Use | Audio Model Adaptation |
|-----------|---------|----------------------|
| **QLoRA** | Quantize 7B+ weights to 4-bit, train adapters | Quantize activations to INT8, use 8-bit optimizers |
| **Unsloth** | Fused Triton kernels for attention | Fused kernels for gated TCN blocks + attention pooling |
| **DeepSpeed** | ZeRO for multi-GPU weight sharding | ZeRO-Offload for larger datasets, gradient checkpointing |

---

## Phase 1: Memory Optimization (QLoRA-inspired)

### 1.1 8-bit AdamW Optimizer
**File:** `train.py`
- Replace `torch.optim.AdamW` with `bitsandbytes.optim.AdamW8bit`
- Cuts optimizer state from ~17MB (fp32) to ~4MB (8-bit)
- Paged optimizer prevents OOM spikes (QLoRA's paged optimizers)

```python
# Current
self.optimizer = torch.optim.AdamW(self.model.parameters(), lr=2e-4)

# New
import bitsandbytes as bnb
self.optimizer = bnb.optim.AdamW8bit(self.model.parameters(), lr=2e-4)
```

### 1.2 Activation Quantization (INT8)
**File:** `model_tcn.py` — `FrequencyAwareEncoder.forward()`
- Quantize intermediate activations in TCN stacks to INT8
- Dequantize only at skip connections and output
- Saves ~50% activation memory during backward pass

### 1.3 Gradient Checkpointing
**File:** `model_tcn.py` — `FrequencyPreservingTCN`
- Wrap TCN stack forward passes with `torch.utils.checkpoint.checkpoint()`
- Trades compute for memory — recomputes activations during backward
- Useful when scaling to larger models or longer audio sequences

## Phase 2: Kernel Optimization (Unsloth-inspired)

### 2.1 Fused Gated Activation Kernel
**File:** `model_tcn.py` — `GroupedGatedBlock`
- Current: 3 separate ops (tanh, sigmoid, multiply) → 3 memory reads/writes
- New: Single fused Triton kernel
- Expected: 2-3x faster TCN blocks, 75% less memory traffic

```python
# Fused operation: tanh(filter_conv(x)) * sigmoid(gate_conv(x))
# Single kernel replaces 3 separate CUDA kernels
```

### 2.2 Fused Attention Pooling Kernel
**File:** `model_tcn.py` — `AttentionTemporalPool`
- Fuse: matmul → scale → softmax → matmul
- Single kernel for the entire attention mechanism

### 2.3 Fused 2D Conv + BN + GELU
**File:** `model_tcn.py` — `SpectralConvBlock2D`
- Current: Conv2d → BatchNorm → GELU (3 kernels)
- New: Fused kernel for the entire block

## Phase 3: Distributed Training (DeepSpeed-inspired)

### 3.1 DeepSpeed ZeRO Integration
**File:** `train.py` — `Trainer` class
- Add DeepSpeed config for ZeRO-2 (optimizer + gradient sharding)
- Enables larger effective batch sizes on multi-GPU setups
- Config file: `conf/deepspeed_config.json`

```json
{
  "zero_optimization": {
    "stage": 2,
    "offload_optimizer": {
      "device": "cpu",
      "pin_memory": true
    }
  },
  "bf16": { "enabled": true },
  "gradient_accumulation_steps": 4
}
```

### 3.2 ZeRO-Offload for Large Datasets
- When dataset grows beyond VRAM (e.g., MUSDB18 full training)
- Offload optimizer states to CPU RAM
- "Emergency button" for OOM errors — slower but prevents crashes

### 3.3 Multi-GPU DDP Support
- Add `torch.distributed` initialization
- Wrap model with `DistributedDataParallel`
- Shard DataLoader across GPUs

## Phase 4: Training Pipeline Improvements

### 4.1 Lazy Dataset Loading
**File:** `dataset.py`
- Current: All 10K samples precomputed in RAM (~2.6GB)
- New: Streaming mode that generates mel-spectrograms on-the-fly
- Reduces RAM footprint for larger datasets

### 4.2 Pin Memory + Async GPU Transfer
**File:** `train.py` — DataLoader
- Set `pin_memory=True` for async GPU transfer
- Add `prefetch_factor=2` for data loading overlap

### 4.3 torch.compile Integration
**File:** `train.py` — after model creation
- `self.model = torch.compile(self.model)`
- Graph-level optimization, 1.5-2x speedup expected
