## Remediation Plan (Prioritized)

### Phase 1: CRITICAL — Add Early Stopping + Checkpoint Selection

**Problem:** Best checkpoint was epoch 9, but training ran 80 epochs wasting compute and the final model is 14% worse than the best.

**Implementation: Add early stopping to `train.py`**

```python
# In Trainer.__init__:
self.patience = 15
self.patience_counter = 0
self.best_val_loss = float('inf')

# In fit(), after validation:
if val_loss < self.best_val_loss - 0.1:  # 0.1 dB min improvement
    self.best_val_loss = val_loss
    self.patience_counter = 0
    self._save_checkpoint(epoch, val_loss, metrics, is_best=True)
else:
    self.patience_counter += 1
    if self.patience_counter >= self.patience:
        print(f"  [early_stop] No improvement for {self.patience} epochs. Stopping.")
        break
```

### Phase 2: CRITICAL — Reduce Model Capacity

**Problem:** 1.8M params on 10K samples (180 params/sample) is too much. The model memorizes training data.

**File: `conf/config.yaml`**
```yaml
model:
  encoder:
    channels: 64        # Reduce from 128 (4x fewer params)
    num_blocks: 3       # Reduce from 4
    num_stacks: 2       # Keep
    embedding_dim: 64   # Reduce from 128
```

Expected param count: ~200K-300K (down from 1.8M), ratio ~25 params/sample.

### Phase 4: HIGH — Add SpecAugment + Noise Injection

**Problem:** Only augmentation is random volume scaling. Need spectral masking for robustness.

**File: `train.py` — add to `train_one_epoch`, after `_prepare_input`:**

```python
# SpecAugment
B, n_mels, T = mel_frames.shape
# Frequency masking
for _ in range(2):
    f = torch.randint(0, 15, (1,)).item()
    f0 = torch.randint(0, n_mels - f, (1,)).item()
    mel_frames[:, f0:f0+f, :] = mel_frames.mean(dim=-1, keepdim=True).expand_as(
        mel_frames[:, f0:f0+f, :])
# Time masking
for _ in range(3):
    t = torch.randint(0, 30, (1,)).item()
    t0 = torch.randint(0, T - t, (1,)).item()
    mel_frames[:, :, t0:t0+t] = mel_frames.mean(dim=-2, keepdim=True).expand_as(
        mel_frames[:, :, t0:t0+t])
# Gaussian noise
mel_frames = mel_frames + torch.randn_like(mel_frames) * 0.02
```

### Phase 5: MEDIUM — Tune Loss Weights for Q Learning

**Problem:** Q MAE is flat at 0.465 decades. The Q head isn't learning.

**File: `conf/config.yaml`**
```yaml
loss:
  lambda_q: 3.0          # Increase from 1.5 (Q learning is stalled)
  lambda_gain: 1.0       # Reduce from 1.5 (gain is learning OK)
  lambda_type: 5.0       # Increase from 3.0 (type stuck at 55%)
  lambda_hmag: 1.0       # Increase from 0.5 (drives spectral shape learning)
```

### Phase 6: MEDIUM — Switch to ReduceLROnPlateau

**Problem:** CosineAnnealing doesn't react to validation performance. The LR keeps dropping even when val loss is increasing.

**File: `train.py`**
```python
# Replace CosineAnnealingLR with ReduceLROnPlateau:
self.scheduler = torch.optim.lr_scheduler.ReduceLROnPlateau(
    self.optimizer, mode='min', factor=0.5, patience=5, min_lr=1e-6
)
# In fit(): call self.scheduler.step(val_loss) instead of self.scheduler.step()
```

### Phase 7: LOW — Increase Dropout Further

**File: `differentiable_eq.py`**

self.dropout = nn.Dropout(p=0.3)  # Increase from 0.2

## Recommended Execution Order

1. **Add early stopping** (Phase 1) — prevents wasted compute, always beneficial
2. **Reduce model capacity** (Phase 2) — addresses root cause of overfitting
3. **Increase dataset to 30K** (Phase 3) — more data for the smaller model
4. **Add SpecAugment** (Phase 4) — regularization through augmentation
5. **Switch to ReduceLROnPlateau** (Phase 6) — adaptive LR for val-based optimization
6. **Tune loss weights** (Phase 5) — improve Q and type learning
7. **Increase dropout** (Phase 7) — last resort if overfitting persists

## Expected Outcome

With a 300K-param model, 30K MUSDB18 samples, SpecAugment, and early stopping:
- **Val loss:** Should reach ~28-30 (vs current best 31.88)
- **Gain MAE:** Should reach ~4.0 dB (vs current 5.02)
- **Type accuracy:** Should reach 60-65% (vs current 55%)
- **Training time:** Should stop at epoch 15-25 (vs 80 wasted epochs)
